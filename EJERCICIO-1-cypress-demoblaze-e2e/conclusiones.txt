2. EJERCICIO - AUTOMATIZACIÓN E2E
================================================================================

2.1 FLUJO IMPLEMENTADO:
-----------------------
1. Agregar dos productos al carrito (Samsung galaxy s6 y Nokia lumia 1520)
2. Visualizar el carrito y verificar productos agregados
3. Completar el formulario de compra con datos de prueba
4. Finalizar la compra y validar el mensaje de confirmación

2.2 TEST CASES:
---------------
- TC001: Flujo completo de compra exitoso
- TC002: Validación del cálculo correcto del total del carrito
- TC003: Funcionalidad de eliminar productos del carrito

2.3 PRINCIPALES HALLAZGOS:
--------------------------
• Los tiempos de carga del sitio pueden variar, por lo que se requirió el uso de esperas explícitas (cy.wait)
• Las alertas JavaScript generadas al agregar productos fueron manejadas correctamente con cy.on('window:alert')
• El mensaje de confirmación de compra utiliza Sweet Alert (selector '.sweet-alert')
• Es necesario retornar manualmente al home después de agregar productos para continuar el flujo

2.4 MEJORES PRÁCTICAS IMPLEMENTADAS:

5. DESAFÍOS Y SOLUCIONES
================================================================================

DESAFÍO 1: Tiempos de carga inconsistentes  
Solución: Uso estratégico de cy.wait() combinado con validaciones mediante .should()

DESAFÍO 2: Manejo de alertas JavaScript  
Solución: Captura y validación de alertas utilizando cy.on('window:alert')

DESAFÍO 3: Navegación no automática tras ciertas acciones  
Solución: Control explícito de la navegación mediante cy.visit('/') para asegurar continuidad del flujo
